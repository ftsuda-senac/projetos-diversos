Exemplo de upload

Conceitos

  • Lembrar que o upload em um sistema web é essencialmente o envio de um arquivo da sua máquina (através do navegador) para o servidor remoto.
  • Na tela HTML com o formulário de envio, lembrar de configurar na tag <form> o atributo enctype="multipart/form-data"
  • O Servlet é o componente da aplicação no servidor que vai pegar o arquivo recebido pela rede e salvar em algum diretório no servidor. A lógica envolve basicamente:
    1. Receber os bytes do arquivo q foi recebido da rede e
    2. Copiá-los no diretório/arquivo de destino.
  • Como o arquivo foi salvo no servidor, será necessário configurar uma URL para que o navegador possa acessá-lo através da web. Isso será feito através de uma configuração diretamente no Tomcat seguindo as instruções abaixo

Instruções para uso no Windows e servidor Tomcat:

  • Criar o diretório C:\uploads onde os arquivos serão salvos após o upload.
    • Se quiser alterar o diretório, alterar o valor no @WebInitParam no UploadServlet
  • Configurar o Tomcat para criar uma raíz de contexto para acessar o conteúdo do diretório C:\uploads através da Web.
    • Editar o arquivo server.xml do Tomcat: pelo Netbeans, na aba serviços acessar a opção "Servidores", escolher o Tomcat, clicar com o botão direito e selecionar opção "Editar server.xml".
    • No arquivo, adicionar o trecho <Context docBase="C:/uploads" path="/teste-uploads" /> antes de fechar a tag </Host> e salvar.
  • Reiniciar o Tomcat, acessar o link /upload e testar.
